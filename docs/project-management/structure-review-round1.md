## 3. 構成上の気になる点（候補）

### 3.1 章タイトル表記の不一致（ナビゲーション上の混乱リスク）
- 複数章で、YAML front matter の `title` と本文の見出し（H1）が一致していません。
  - 例: 第1章 `title: 第1章: ファイル操作の基本` / H1: `第1章: Linuxの世界への第一歩`
- 目次（`docs/index.md`）の表記とも揺れが出るため、読者の「今どの章を読んでいるか」認知に影響します。

### 3.2 第0章のみ「まとめ」見出しが無い
- 第1〜6章は `## まとめ` があり、章末の到達点が明確です。
- 第0章は `🎉 セットアップ完了！` で締まるため実用上は問題ありませんが、「章テンプレートの統一」という観点では `## まとめ` を追加する余地があります。

## 4. 構成改善の提案（案）

### 4.1 `title` / H1 / 目次の章タイトルを統一
- 原則として、以下を一致させる（提案）:
  - YAML front matter の `title`
  - 章ページの H1
  - `docs/index.md` の目次表記
- これにより、ナビゲーション表示や章間リンクの表記ゆれを解消できます。

### 4.2 第0章にも `## まとめ` を追加（任意）
- 第0章の末尾に、達成条件（例: `wsl --install` 実行済み、Ubuntu起動、基本コマンドで疎通確認）を短い箇条書きで置く。
- 既存の「次章へ」リンクは維持する。
